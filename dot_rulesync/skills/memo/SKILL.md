# メモ書き出しスキル

会話中に行った PR レビュー・設計調査・アーキテクチャ分析の結果を、プロジェクトの `10-memo-*.md` ファイルに加筆する。

## 手順

### 1. 加筆先ファイルの特定

ユーザーが加筆先を指定している場合はそのファイルを使う。指定がない場合は以下の順で探す:

1. プロジェクトルートの `10-memo-<リポジトリ名>.md` を Glob で探す
2. 見つからなければユーザーに確認する

### 2. 既存ファイルを必ず Read する

**ファイルを Read せずに Edit / Write しない。** 既存の内容・目次・セクション番号を把握してから追記する。

### 3. 目次にエントリを追加する

既存の目次がある場合、新しいセクション番号とリンクを追加する。

例：
```md
## 目次

1. [既存のセクション](#1-既存のセクション)
2. [新しいセクション](#2-新しいセクション)  ← 追加
```

### 4. 末尾に本文セクションを追記する

以下の構成でセクションを書く:

```md
---

# N. セクションタイトル

## N-1. 背景・概要

...

## N-2. 詳細

...

## N-3. 推奨・まとめ

...
```

## 注意事項

- コードスニペットは「現状」「比較」「推奨」の3点セットで書くと再利用しやすい
- 「なぜそうなっているか（背景・理由）」を必ず含める
- 「次にどうすべきか（推奨）」を短期・長期に分けて書く
- 一時的なタスク固有の情報（特定のバグ詳細など）は含めない
