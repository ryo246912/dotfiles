# Claude Features & Usage Guide (Q1 2026)

Anthropicが2026年第1四半期にリリースした120以上の機能のうち、特に重要な「Sティア」機能の設定方法と使い方をまとめます。

## 1. Auto Mode (自動実行モード)

Claudeがユーザーに代わって許可判断を行い、プロンプトによる中断を最小限にします。

- **設定方法:**
  ```bash
  claude --enable-auto-mode
  ```
- **特徴:**
  - 全ての操作の前に2つの安全チェックを実行します。
  - ファイルの編集やテストの実行を自動で行います（データベースの削除などの危険な操作は行いません）。
- **ユースケース:** 大規模なリファクタリングや、複数のファイルにまたがる修正など、従来なら数十回の許可クリックが必要だった作業。

## 2. Opus 4.6

2026年2月にリリースされた最新モデル。

- **特徴:**
  - タスクの持続時間（Task Horizon）が大幅に向上し、**最大14時間**一つのタスクを追跡可能。
  - 後述する `/loop` や Agent Teams の基盤となります。

## 3. Agent Teams (エージェント・チーム)

リードエージェントが最大10人の並列チームメイトにタスクを委譲し、共同作業を行います。

- **設定方法 (環境変数):**
  ```bash
  export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
  ```
- **特徴:**
  - 複数の課題（例：認証フローの修正、ダッシュボードのレイアウト変更、CSV出力の修正など）を同時に指示できます。
  - チームメイト同士が調整し合いながら並列でコードを書き、PRを作成します。
- **要件:** Opus 4.6 が必要（現在はリサーチプレビュー版）。

## 4. Cowork

コンピュータ上で永続的に実行され、アプリケーション、ファイル、画面にアクセスできる機能です。

- **使い方:**
  - 会議中などにバックグラウンドで作業を依頼できます。
  - 例：「SlackのDMを確認して要約して」「Driveにある売上チャートを最新化して」
- **メリット:** ユーザーが席を外している間に、Slackの要約や資料の更新などのルーチンワークを完了させます。

## 5. Connectors (コネクタ)

Gmail, Slack, Notion, Figma, Asana, Google Driveなど、50以上のツールとClaudeをワンクリックで連携させます。

- **特徴:**
  - MCP (Model Context Protocol) をベースにしていますが、ユーザーはMCPを意識せずにトグルスイッチで設定可能。
  - **Figma連携の例:** デザインのモックアップからReactコンポーネントを生成し、コードの調整内容をFigma側に書き戻す双方向のループが可能。

## 6. /loop (ループ・コマンド)

セッション内でタスクを定期的に繰り返し実行します。

- **使用例:**
  ```bash
  /loop 5m /babysit        # 5分ごとにコードレビューに対応しPRをプッシュ
  /loop 30m /slack-feedback # 30分ごとにSlackフィードバック用にPRを作成
  /loop 1h /pr-pruner      # 1時間ごとに古いPRをクローズ
  ```
- **特徴:**
  - 最大7日間実行可能。
  - 1セッションあたり最大50タスクをサポート。
  - 他のスラッシュコマンド（スキルなど）と組み合わせてワークフローを自動化できます。

---

## 始め方のアドバイス

1. **Auto Modeをオンにする:** 許可クリックの手間を省く。
2. **Coworkをセットアップする:** 会議中などの空き時間を活用する。
3. **`/loop 5m /babysit` を試す:** 次のプルリクエスト作成時に自動化を体感する。
