# Gemini Code Assist Review Guide

## 出力言語

- レビューコメント、PR 概要、要約は常に日本語で記述してください。

## レビュー方針

- 指摘は根拠ベースで簡潔にまとめ、推測を含む場合は推測であることを明示してください。
- dotfiles / chezmoi リポジトリとして、既存ユーザー環境を壊す変更、OS 依存パス、権限差異、破壊的なコマンドを優先して確認してください。
- 既存コメントや設定の意図を尊重し、キー順やフォーマットだけを理由にした不要な churn は避けてください。
- セキュリティ、秘密情報、プライバシー、テレメトリ設定の変更は優先して確認してください。
- 大きな設定ダンプや生成物に対する些細な指摘は避け、保守性、安全性、再現性に直結する点に絞ってください。
- VS Code 設定や JSONC ではコメント保持を優先し、コメント削除だけを目的とした提案は避けてください。
- シェルやワークフロー定義では、引用、エラーハンドリング、冪等性、macOS / Linux 差異に注意してください。
