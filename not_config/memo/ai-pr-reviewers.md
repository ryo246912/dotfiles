# AIプルリクエスト（PR）コードレビューサービスの調査報告

CodeRabbit以外のAIコードレビューサービスについて、無料で利用できるものを中心に調査した結果をまとめます。

## 1. サービス比較一覧

| サービス名 | 特徴 | 無料枠の内容（非公開リポジトリ） | 導入方法 | 推奨度 |
| :--- | :--- | :--- | :--- | :--- |
| **PR-Agent (Qodo)** | オープンソース。対話型レビューに強い。 | **完全無料** (セルフホスト時) | GitHub Action / App / CLI | ★★★★★ |
| **Ellipsis (YC W24)** | 高品質・低ノイズ。学習機能あり。 | 1ユーザーまで（個人・OSS向け） | GitHub App | ★★★★☆ |
| **DeepSource** | 静的解析 + AI。修正案の自動生成。 | 3ユーザーまで | GitHub App | ★★★★☆ |
| **Trunk Check** | Linter/Formatter統合型。 | 5コミッターまで | GitHub App / CLI | ★★★☆☆ |
| **Bito AI** | セキュリティとプライバシー重視。 | 個人利用は無料 | IDE / GitHub App | ★★★☆☆ |

## 2. 各サービスの詳細

### **PR-Agent (CodiumAI / Qodo Merge)**
- **概要:** オープンソース（Python）として公開されており、`/review`, `/describe`, `/improve`, `/ask` などのコマンドでAIと対話しながらレビューを進められます。
- **無料運用のコツ:** 自分のOpenAIやAnthropicのAPIキーを使用し、**GitHub Actions**で動かす構成にすれば、サービス利用料は一切かかりません（API実費のみ）。
- **メリット:**
    - プロンプトを詳細にカスタマイズ可能。
    - ワークフローに組み込みやすく、追加のSaaSアカウントが不要。
- **デメリット:** YAML設定やAPIキー管理など、導入に若干のエンジニアリングが必要。

### **Ellipsis (YC W24)**
- **概要:** 「技術リーダーのようなレビュー」を目指しており、非常にノイズ（些細な指摘）が少ないのが特徴です。
- **メリット:**
    - 指摘を却下するとAIがそれを学習し、次回から同様の指摘を抑制。
    - コメントが具体的でアクションにつながりやすい。
- **デメリット:** 多人数チームや大規模開発では有料プランへの移行が必要。

### **DeepSource**
- **概要:** 従来の静的解析（Linter）とAIを高度に統合したツールです。
- **メリット:**
    - 指摘箇所に対して「Autofix」ボタンが表示され、クリックするだけで修正コミットを自動作成。
    - セキュリティ脆弱性の検知に強い。
- **デメリット:** AIとの対話よりも、ルールベースの自動修正に重きを置いている。

### **GitHub Copilot (PR Review機能)**
- **概要:** 既にGitHub Copilotを導入している場合、追加費用なしで利用可能。
- **特徴:** PRの変更内容の要約作成には優れているが、詳細なロジックレビューは特化型ツールに一歩譲る。

## 3. 推奨導入プラン：PR-Agent (GitHub Actions)

完全に無料で柔軟に運用したい場合は、PR-AgentをGitHub Actionsで運用するのが最適です。

### 導入手順概要:
1. GitHubリポジトリの `Settings > Secrets` に `OPENAI_KEY` を登録。
2. `.github/workflows/pr_agent.yml` を作成し、公式アクションを呼び出す。

---

## 4. 結論
- **品質とスマートさ重視:** **Ellipsis** の無料枠を推奨。
- **完全無料・自由度重視:** **PR-Agent (GitHub Actions)** で自前構築。
- **自動修正まで行いたい:** **DeepSource**。
